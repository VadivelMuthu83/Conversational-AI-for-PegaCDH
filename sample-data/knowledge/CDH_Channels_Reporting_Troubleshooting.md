# Pega CDH — Channels, Reporting & Troubleshooting Guide

## Channel Configuration in Pega CDH

### Supported Channel Types

| Channel | Type | Direction | Real-Time | Use Case |
|---------|------|-----------|-----------|----------|
| Web | Digital | Inbound | Yes | Portal personalisation, product pages |
| Mobile | Digital | Inbound | Yes | App personalisation, push triggers |
| Email | Digital | Outbound | No (batch) | Campaign sends, triggered emails |
| SMS | Digital | Outbound | Near-real-time | Short notifications, alerts |
| Push | Digital | Outbound | Near-real-time | Mobile app push notifications |
| CallCentre | Assisted | Inbound | Yes | Agent desktop recommendations |
| CallCentre | Assisted | Outbound | Yes | Outbound calling campaigns |
| Branch | Physical | Inbound | Yes | Teller/advisor desktop |
| ATM | Physical | Inbound | Yes | ATM screen personalisation |
| Print | Physical | Outbound | No (batch) | Statement inserts, letters |

### Channel Configuration Path

```
CDH > Channels > [Channel Name]
  - Channel type and direction
  - Response window (how long to wait for outcome)
  - Batch vs real-time processing
  - Serialisation (max actions per interaction)
  - Contact policy integration
```

### Real-Time vs Batch Channels

**Real-time channels** (Web, Mobile, CallCentre, Branch):
- CDH decision must complete in < 500ms
- Uses in-memory propensity caching
- ADM model updates happen asynchronously
- Returns top N actions immediately

**Batch channels** (Email, Print):
- CDH pre-computes decisions for target population
- Runs as scheduled job (typically nightly)
- Decisions queued for delivery system
- ADM updates run after outcomes received

## Channel Effectiveness Analysis

### Key Metrics by Channel

Track these per channel per week:

| Metric | Formula | Good | Alert If |
|--------|---------|------|----------|
| Impressions | COUNT(*) WHERE Channel = X | — | Sudden drop |
| Accept Rate | Accepts / Impressions | > 5% | < 2% |
| Avg Propensity | AVG(Propensity) | > 0.3 | < 0.15 |
| Response Rate | Non-Impression outcomes / Impressions | > 60% | < 30% |
| Revenue/Impression | Revenue / Impressions | > £5 | < £1 |

### Channel Comparison Table (Example)

| Channel | Impressions | Accept Rate | Avg Propensity | Revenue/Impression |
|---------|-------------|-------------|----------------|-------------------|
| Web | 45,231 | 8.2% | 0.41 | £12.40 |
| Mobile | 23,118 | 9.7% | 0.44 | £14.20 |
| Email | 89,445 | 4.1% | 0.32 | £6.80 |
| SMS | 12,334 | 3.8% | 0.29 | £4.20 |
| CallCentre | 8,231 | 15.3% | 0.52 | £28.50 |
| Push | 34,112 | 2.9% | 0.27 | £3.10 |

**Insight from this table:**
- CallCentre has highest accept rate (15.3%) and revenue/impression (£28.50) but lowest volume
- Push has lowest accept rate and revenue — review if actions are appropriately targeted
- Mobile outperforms Web slightly — consider shifting budget to mobile

## Troubleshooting Common CDH Issues

### Issue 1: Accept Rate Dropped Suddenly

**Symptoms:** Overall accept rate fell > 15% week-over-week.

**Diagnostic steps:**
1. Check if the drop is across all actions or specific ones
2. Review strategy change log — did any weights/values change?
3. Check ADM model AUC — did any model AUC drop?
4. Check IH outcome pipeline — are outcomes being captured?
5. Review contact policy — were limits tightened recently?

**Common causes and fixes:**

| Root Cause | Fix |
|-----------|-----|
| Weight reduced on high-performing action | Revert weight change |
| ADM model reset (lost training) | Allow model to retrain — 4-6 weeks for recovery |
| Outcome pipeline stopped | Fix pipeline; recent outcomes all show as Impression |
| Contact policy tightened | Review if tightening was intentional |
| Campaign ended | Normal if campaign-driven action was driving volume |

### Issue 2: Specific Action Never Selected

**Symptoms:** Action has impressions = 0 despite being in active strategy.

**Diagnostic steps:**
1. Decision Audit for a customer who should have received it
2. Check eligibility rules — are any blocking all customers?
3. Check applicability rules — is the channel wrong?
4. Check weight — is it set to 0?
5. Check active date range — has the action expired?

```
CDH > Tools > Decision Audit
Customer: [Any active customer]
Date: Today
Look for action in "Actions considered" list
If not shown: Eligibility failed
If shown but not selected: Check priority vs winning action
```

### Issue 3: ADM Model Not Learning (AUC Flat)

**Symptoms:** Model AUC stays at 0.5–0.55 after > 1,000 responses.

**Root causes:**

| Cause | Diagnosis | Fix |
|-------|-----------|-----|
| No positive outcomes | Check Positivity rate in snapshot | Verify outcome pipeline is capturing accepts |
| Predictors not reaching model | Check ActivePredictors count | Review data flow mapping |
| Outcome definition wrong | Check what counts as positive | Align outcome definition with ADM |
| Model reset too recently | Check model age vs response count | Allow more data to accumulate |

### Issue 4: IH Growing Too Fast

**Symptoms:** IH database growing faster than expected; performance degrading.

**Diagnostic:**
1. Check average impressions per customer per day
2. Identify top channels by volume
3. Check if contact policy is configured
4. Check for runaway batch job sending millions of impressions

**Fix options:**
- Implement or tighten contact policy
- Set IH retention to 90 days (purge older records)
- Archive IH older than 90 days to cold storage
- Check batch channel job for misconfiguration

### Issue 5: Value Finder Shows No Underserved Customers

**Symptoms:** All customers appear in Balanced or Lost quadrant.

**Diagnostic:**
1. Check Value Score configuration — are values non-zero?
2. Check Engagement Score — is IH data flowing into Value Finder?
3. Check segmentation — are segments too broad?

**Fix:**
- Verify Value Components are configured for each action
- Ensure IH data flow is connected to Value Finder
- Review segment granularity — use finer segments for better insight

## CDH Environment Management

### Environment Types

| Environment | Purpose | Data | Real Customers |
|-------------|---------|------|----------------|
| Development | Strategy authoring | Synthetic | No |
| Test/QA | Strategy testing | Anonymised IH copy | No |
| Staging | Pre-prod validation | Production clone | No |
| Production | Live operation | Real IH | Yes |

### Promoting Strategy Changes

```
Development → Test: Developer self-service
Test → Staging: Requires test sign-off
Staging → Production: Requires:
  - Strategy manager approval
  - Compliance sign-off (if Suitability rules changed)
  - Impact Analyzer results attached
  - Deployment window approved (avoid peak hours)
```

### Deployment Windows

Recommended deployment schedule:
- **Web channel changes:** Tuesday–Thursday 10pm–6am
- **Email campaign changes:** Any time (batch, no real-time impact)
- **ADM model changes:** Sunday 2am–4am (lowest traffic)
- **Contact policy changes:** First day of month (clean period boundaries)

## CDH Security and Access Control

### Role-Based Access

| Role | Permissions |
|------|------------|
| Strategy Designer | Create/edit engagement policy, weights, actions |
| Strategy Manager | Approve and deploy strategy changes |
| Data Analyst | Read-only access to IH, reports, exports |
| Compliance Officer | Read IH, Explainability, Contact Policy |
| System Administrator | All configuration, environment management |
| Business User | View dashboards only |

### Data Security for IH

- IH contains personal data — subject to GDPR/CCPA
- Mask CustomerID in non-production environments
- Restrict IH export to authorised roles
- Log all IH exports in audit trail
- Customer data deletion requests must purge IH records
  ```
  CDH > Data Management > Customer Data Deletion
  Enter: CustomerID
  Action: Remove all IH records for this customer
  ```
